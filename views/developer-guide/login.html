Login

Para desenvolver aplicativos que resgatam dados do usuário, é preciso que os usuários tenham dado permissão a sua aplicação para acessar os dados dele. Nesse caso, o usuário precisa se identificar e conceder essa permissão. Para isso, a Ingresse desenvolveu um mecanismo de identificação próprio, evitando que aplicativos de terceiros manipulem os dados de acesso a conta de nossos usuários.

Para esses métodos você precisa enviar via GET o usertoken. O usertoken é gerado pela Ingresse e pode ser armazenado pela sua aplicação associando ao mesmo usuário.

Para solicitar o usertoken faça uma requisição usando o método /authorize. Essa requisição irá retornar a página HTML de login de usuário da Ingresse. Através dessa página o usuário irá informar seu email e senha autorizando que o respectivo aplicativo acesse seus dados da Ingresse. Por padrão, o retorno desse login é um JSON com a seguinte estrutura:

{token: value, userId: value}

Porém, na requisição do método /authorize você pode enviar também via GET o parâmetro returnurl com a URL de retorno no sucesso do login. Dessa forma, no sucesso do login a API irá redirecionar o resultado para a URL em returnurl enviando via GET as mesmas informações acima, isto é, token e userId.
