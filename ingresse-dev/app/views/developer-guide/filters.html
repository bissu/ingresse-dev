Filtros

Tornamos a utilização da API Ingresse flexível com a aplicação de filtros de conteúdo. Esses filtros permitem que sua aplicação manipule como e o que irá ver de cada conteúdo. Para utilizar os filtros é simples, basta adicionar na url o manipulador que deseja utilizar e a API se encarrega do resto. Abaixo seguem a lista de filtros já disponíveis.

Na página de cada método, ou categoria de método, é possível ver os filtros disponíveis para cada método.

Paginação

Acessando uma página

Para acessar páginas de listas de conteúdos, basta acrescentar na url o número da página que deseja acessar através da variável page. Por exemplo, para ver a página 2 da lista de eventos, use: api.ingresse.com/event?page=2.

Para saber quantas páginas existem na lista, procure pelo campo lastPage, dentro do campo paginationInfo no JSON de resposta da lista solicitada.

Exemplo de retorno:

// Início de JSON de resposta
{
	"responseData":{
		"paginationInfo":{
			"currentPage":1,
			"lastPage":7,
			"totalResults":154,
			"pageSize":25
		},
... // JSON continua

Definindo tamanho da página

Para aumentar ou diminuir o tamanho da página que está sendo retornada, utilize a variável pagesize e defina a quantidade de items que deseja. Por exemplo, para trazer 50 eventos por página na lista de eventos, use: api.ingresse.com/event?pagesize=50.

Ao fazer isso, os dados de quantidade de páginas e tamanho da página estarão alterados.

Filtros de atributos

É possível espeficiar que informações deseja trazer de cada entidade (eventos, usuários, ingressos etc). Para isso, acrescente o filtro fields e logo após adicione o nome dos campos que deseja que venham. Os itens devem vir separados por vírgula e escritos de acordo com os atributos da entidade de retorno. No caso de uso do filtro de campos, automaticamente todos os atributos não mencionados no item serão removidos da chamada.

Exemplo, para trazer a lista de eventos apenas com título e data, faça: api.ingresse.com/event?fields=title,date
